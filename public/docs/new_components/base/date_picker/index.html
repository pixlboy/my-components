Explanation

Goal of Component: There are many places that require data picker. This component aims at providing belowing features:

  1. User can have different input and output date format, this can suite various application requirements
  2. Input and output can be either timestamp or date string
  3. Timezone support is also provided

Usage of Component: 

				var onDateChange = function(date, ts){
					document.getElementById('result').innerHTML = "Selected Results: " + date +", ts:"+ts;
				}
				var DatePicker = JnprCL.DatePicker;
				ReactDOM.render(React.createElement(DatePicker, {
					onDateChange: this.onDateChange,
					format: 'DD-MMM-YYYY',
					timezone: 'PST',
					minDate: 1519691729000,
					date_ts: '28-Feb-2018' // 1519691729000
				}), document.getElementById('output'));
Meaning for properties:
  1. format (optional): default is 'YYYY-MM-DD'. List of format is as below:
    • DD - two digits days
    • MM - two digits months
    • MMM - three characters for months(short)
    • YYYY - four digits for year
    Now, you can use separators, such as '-' or '/' to separate different parts of the date
  2. timezone(optional): default is 'PST'. This timezone is the same as dataTable requirements
  3. date_ts(optional): default value for date or timestamp. Here, it can be either string (such as 2018-01-01) or timestamp (in milliseconds)
  4. onDateChange: callback handler whenever user changed the date.It accept two parameters, first one is string format and second parameter is timestamp. Please take whatever you need.
  5. minDate: Minimum date to be able to select. Please use timestamp here.
  6. In case you need to set up date from external, please use store like below: 
    						var setDate = function(){
							var Store = JnprCL.DatePickerStore;
							Store().setNewDate(	document.getElementById('txtExternalDate').value  );
						}
